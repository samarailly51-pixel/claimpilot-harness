# Community Case Pack

ClaimPilot grows through small, reviewable, fictional claim scenarios. A useful first contribution is one case that exposes one clear failure mode.

## Contribution path

1. Open the `Good first case contribution` issue template.
2. Claim one scenario or propose a new claim line.
3. Copy `cases/template-case.json` and add stable evidence IDs.
4. Run validation, catalog, suite, and tests locally.
5. Open a pull request with the generated replay or score summary.

Good cases force an Agent to slow down because evidence conflicts, a required document is missing, a policy condition is unclear, or uploaded content is adversarial.

Never include real claimant information. Names, policy numbers, dates, medical details, invoices, and employer records must be fictional or synthetic.
